# Contributing to zed-moonbit

Thank you for your interest in contributing! This project aims to be a clean, simple, and maintainable base for integrating MoonBit into Zed.

## How to Propose Changes

- We invite you to open an issue to discuss a bug or propose a new feature before starting to write code.
- Fork the repository and create a concise dedicated branch.
- Ensure that the changes are small and well-motivated to facilitate review: an isolated fix or a single supported query per Pull Request is ideal.

## Code Style

- **Rust**: Use logical and linear code, without overengineering or excessive abstractions. Comments should only be inserted where they explain the "why" behind counter-intuitive behavior.
- **Configuration Files**: Keep YAML, TOML, and SCM files properly indented and formatted.

## Updating queries / config / doc

- Before updating Tree-sitter queries (e.g., `highlights.scm`), carefully compare them with the preview or ASTs generated by upstream Tree-sitter MoonBit, so as not to pollute the grammar with unstable patterns.
- Always keep the `README.md` updated in case of requirement changes.

Thank you for collaborating on open-source code!
